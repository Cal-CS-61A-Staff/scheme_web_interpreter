Scheme Interpreter 2.0 Preview
********************************************************************************
This is a development build of the new Scheme interpreter version.

Major (Planned) New Features
--------------------------------------------------------------------------------
Synchronous Core Interpreter (w/ optional futures and async/await)
Better Modularization (easier to use only specific parts of the interpreter)
Statically Typed (allows for better tooling, fewer bugs, Dart 2 support)
New Frontend (cleanup old hacks; decouple from interpreter implementation)
Can Be Easily Open Sourced (methods/functions that match pieces of the project
    are called through an interface with an implementation we'll keep private)
...

Component Versions
--------------------------------------------------------------------------------
Frontend: Temporary hack of old frontend onto new interpreter
Core Interpreter: 2.0.0-alpha001
StaffProjectImplementation: 2.0.0-alpha001

Feature Status
--------------------------------------------------------------------------------
New Frontend: Not yet started
cs61a_scheme: Core library except for quasiquoting fully implemented.
    Should otherwise properly implement the student/staff specification.
cs61a_scheme.extra: Async support through AsyncExpressions (basically Futures)
    with async/await support through define-async and lambda-async. Still needs
    non-web-dependent diagramming and event hooks.
cs61a_scheme.web: JS interop mostly complete. Still needs diagramming,
    visualization, graphics, theming, and imports.
Library of Scheme code (can't use until imports are done):
    (demo 'autopair) requires hooks, but should otherwise be reasonable
    (demo 'chess/'paint) require graphics, events, possibly some code changes
    (library 'math/'strings/'timing) should all be fine (probably)
    (library 'music) is already largely unusable on the old interpreter
        (changes to AudioContext resulted in large performance regressions)
        Would require substantial changes due to use of old asynchrony.
